# Security Policy

## Supported Versions

Este proyecto se encuentra en una etapa temprana de desarrollo. La política de soporte se aplica únicamente a la última versión publicada.

| Version   | Supported          |
| --------- | ------------------ |
| 0.1.55    | :white_check_mark: |
| < 0.1.55  | :x:                |

Las versiones anteriores no reciben actualizaciones de seguridad. Se recomienda utilizar siempre la versión más reciente disponible.

---

## Reporting a Vulnerability

Si detectás una vulnerabilidad de seguridad, por favor reportala de forma responsable.

### Cómo reportar

Enviar un correo a:  
**gbarosio@gmail.com**

Incluir en el reporte:

- Descripción clara de la vulnerabilidad  
- Pasos para reproducirla  
- Impacto potencial  
- Versión afectada  
- Cualquier evidencia relevante (logs, capturas, payloads, etc.)

### Qué esperar

- Confirmación de recepción dentro de las 72 horas  
- Evaluación inicial dentro de los 5 días hábiles  
- Comunicación directa si se requiere información adicional  

Si la vulnerabilidad es válida:

- Se priorizará su corrección según severidad  
- Se liberará una nueva versión con el fix  
- Se podrá reconocer al reportante (opcional)  

Si la vulnerabilidad es descartada:

- Se explicarán los motivos de la decisión  

### Alcance

Este proyecto:

- Expone información jurídica estructurada  
- Puede integrarse con modelos de IA (MCP/LLM)  
- Puede ser utilizado en contextos sensibles (legal/compliance)  

Sin embargo:

- No es una fuente oficial de derecho  
- No garantiza exhaustividad ni ausencia de errores  
- No debe ser utilizado como única base para decisiones legales críticas sin validación independiente  

### Divulgación responsable

Se solicita no divulgar públicamente la vulnerabilidad hasta que:

- haya sido evaluada, y  
- exista una corrección disponible o un plan de mitigación  

---

## Notas

Dado que el proyecto evoluciona rápidamente, la superficie de ataque puede cambiar entre versiones. La seguridad depende tanto del código como de cómo se implementa e integra el sistema (por ejemplo: uso de MCP, configuración local, fuentes de datos, etc.).
